# 太空策略射击游戏核心策划

## 游戏定位

本项目是一款浏览器端 2D 太空策略射击游戏。玩家驾驶飞船在开放式太空地图中探索，观察不同星球的威胁范围与攻击方式，自主决定是否进入危险区域并挑战星球母体。

本阶段目标不是完成完整游戏，而是先用无依赖 Canvas 方案搭建清晰、易理解、可直接打开运行的最小玩法原型，避免受 npm registry 访问限制影响。

## 核心玩法循环

1. 玩家在太空地图中移动与观察。
2. 玩家发现星球、休息站、陨石或机关等地图目标。
3. 玩家判断目标的风险与收益。
4. 玩家选择绕行、靠近、挑战或休整。
5. 进入战斗或互动后获得资源、解锁路线或提升武器。
6. 玩家继续探索更高难度区域。

## 星球挑战机制

- 每个星球都有独立属性：实力等级、生命值、攻击范围、攻击方式和奖励。
- 玩家进入星球攻击范围后会触发警告，后续阶段会触发战斗弹幕或防御机制。
- 星球母体是区域挑战核心，击败后可以掉落升级材料、地图情报或通行权限。
- 不同星球应具备不同战斗节奏，例如慢速范围打击、连续激光、追踪飞弹、散射弹幕等。

## 星球攻击范围机制

- 星球攻击范围用圆形区域表现，玩家可以在进入前评估风险。
- 进入攻击范围后，界面显示星球名称、强度和攻击方式提示。
- 后续阶段可以为攻击范围加入状态变化：警戒、锁定、开火、冷却。
- 攻击范围应服务于策略选择，让玩家能够主动决定是否挑战，而不是被动遭遇。

## 小地图机制

- 小地图用于展示玩家当前位置、已发现星球、休息站和危险区域。
- 初期小地图只显示已探索区域，后续可加入扫描器扩大视野。
- 小地图应帮助玩家规划路线，避免迷路，并标记高风险目标。

## 休息站机制

- 休息站是地图中的安全点。
- 玩家可以在休息站恢复生命、补充弹药、保存进度或进行武器升级。
- 部分休息站可以作为剧情节点或商店入口。
- 后续可加入休息站稀缺性，让玩家在冒险深度和安全撤退之间做选择。

## 后续开发阶段

1. **第一阶段：项目框架与最小原型**
   - 原生 HTML / CSS / JavaScript + Canvas 项目原型。
   - 玩家移动、射击、星空背景、星球占位、攻击范围提示、简易小地图和状态显示。
2. **第二阶段：基础战斗**
   - 星球生命值、玩家子弹命中、简单敌方攻击和受伤反馈。
3. **第三阶段：地图探索**
   - 可滚动大地图、相机跟随、小地图和探索标记。
4. **第四阶段：成长系统**
   - 武器升级、资源掉落、休息站恢复和商店。
5. **第五阶段：环境危险**
   - 陨石、跟踪机关、实体撞击和破散弹幕。
6. **第六阶段：技术迁移与内容打磨**
   - 根据玩法稳定程度，选择继续使用 Canvas 或迁移到 Phaser。
   - 更多星球类型、音效、美术资源、关卡节奏和平衡性调整。
