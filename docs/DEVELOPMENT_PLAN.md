# 分阶段开发计划

## 第一阶段：项目框架与最小可运行原型

目标：确认项目能安装、启动和展示最小核心概念。

- 使用原生 HTML / CSS / JavaScript + Canvas，保证无需安装依赖即可运行。
- 建立基础目录结构。
- 创建黑色太空背景和随机星星。
- 创建玩家飞船临时几何图形。
- 支持 WASD / 方向键移动。
- 支持空格发射子弹。
- 创建至少 3 个星球占位圆形。
- 显示每个星球的攻击范围圆圈。
- 玩家进入攻击范围时显示提示文字。
- 添加简易小地图。
- 显示玩家生命值和当前状态。

## 第二阶段：基础战斗验证

目标：让星球挑战具备最小战斗反馈。

- 为星球加入生命值。
- 为玩家子弹加入命中检测。
- 加入星球受击闪烁和血量条。
- 为星球加入一种简单攻击方式。
- 加入玩家生命值与受伤反馈。
- 加入挑战胜利与失败提示。

## 第三阶段：探索地图与小地图

目标：让游戏从单屏原型扩展为可探索地图。

- 扩大世界边界。
- 加入相机跟随玩家。
- 添加小地图 UI。
- 标记玩家、星球、休息站和危险范围。
- 加入已探索 / 未探索区域概念。

## 第四阶段：休息站与成长系统

目标：形成探索后的正反馈。

- 添加休息站实体。
- 支持恢复生命与补给。
- 添加资源掉落。
- 添加基础武器升级。
- 设计简单商店或升级界面。

## 第五阶段：环境危险与机关

目标：提高地图探索过程中的变化和风险。

- 添加陨石漂移与碰撞伤害。
- 添加跟踪机关。
- 添加实体撞击类敌人或障碍。
- 添加破散弹幕攻击。
- 调整不同危险之间的组合方式。

## 第六阶段：技术迁移、内容扩展与体验打磨

目标：让原型接近可试玩版本。

- 评估是否从无依赖 Canvas 原型迁移到 Phaser 或其他游戏框架。
- 增加更多星球类型。
- 增加不同武器路线。
- 添加音效、背景音乐和基础美术。
- 优化 UI、手感和性能。
- 做难度曲线与数值平衡。
